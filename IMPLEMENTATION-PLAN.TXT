4. Per-module backend folder structure
Every domain module under apps/api/src/modules/<name>/ follows the same layout:

modules/<name>/
├── index.ts                 # register routes + DI bindings
├── <name>.routes.ts         # Fastify route definitions (prefix only)
├── <name>.controller.ts     # HTTP in/out, status codes, calls service
├── <name>.service.ts        # business logic, transactions, orchestration
├── <name>.repository.ts     # SQL/Drizzle only — no HTTP
├── models/
│   └── <name>.model.ts      # DB row types / table mappings
├── schema/
│   └── <name>.schema.ts     # Drizzle table defs (if not in packages/db)
├── validators/
│   ├── <name>.create.validator.ts
│   ├── <name>.update.validator.ts
│   └── <name>.query.validator.ts   # list filters, pagination (Zod)
├── dtos/
│   ├── <name>.response.dto.ts      # API response shapes (camelCase)
│   └── <name>.request.dto.ts
├── mappers/
│   └── <name>.mapper.ts            # row ↔ dto
└── __tests__/                      # integration tests per module
Cross-cutting (apps/api/src/):

├── app.ts
├── config/                 # env schema (Zod)
├── plugins/                # auth, tenant, audit, rate-limit
├── middleware/
│   ├── authenticate.ts
│   ├── requirePermission.ts
│   └── tenantContext.ts
├── lib/
│   ├── errors.ts           # AppError hierarchy
│   ├── pagination.ts
│   └── idempotency.ts
└── modules/...
Layer rules (production):

Controller: no SQL; map Zod parse errors → 400.
Service: owns transactions; calls other modules via service interfaces, not repositories directly (except own module).
Repository: parameterized queries only; enforce organization_id on every query.
Validators: all inputs; share list/query schemas with packages/shared.
DTOs: never expose password_hash, secret_key, raw tokens.

7.3 RAG pipeline (production)
Index: knowledge_articles, closed incident summaries, runbooks → chunk → embed → pgvector table document_embeddings (org_id, source_type, source_id, chunk, embedding).
Retrieve: filter by organization_id + user permissions.
Generate: LLM with citations ([KB-123], [INC-1042]).
Audit: every Copilot turn → audit_logs + copilot_messages with model_used, token counts (already in schema).
7.4 Streaming API (replace mock Copilot)
POST /v1/copilot/sessions — create session with workflow_type, optional incident_id.
POST /v1/copilot/sessions/:id/messages — SSE stream tokens (same UX as current typewriter).
Rate limits per org + per user; budget caps in organizations.settings.
7.5 Non-LLM AI (cheaper, always on)
Alert priority score: weighted features (severity, asset criticality, MITRE match, dedup_count) — run synchronously on alert create; optional nightly recalibration.
Event stream anomalies: rolling baselines in Redis → flag → optional LLM explanation async.
7.6 Safety & compliance (must-have for prod)
Redact secrets/PII from prompts (regex + allowlist fields from raw_data).
Human-in-the-loop for destructive tools (isolate host, disable user).
Data residency: org-level model routing in organizations.settings.
Retention: purge copilot_messages per data_retention_days.
8. Auth, RBAC, multi-tenancy
Align API with src/lib/rbac.ts:

JWT claims: sub, org_id, role, permissions[].
Middleware requirePermission('view:events') matches seed roles in schema.
Postgres RLS: USING (organization_id = current_setting('app.current_org')::uuid).
API keys (api_keys): scope checks per route; hash only stored (key_hash, key_prefix).
SSO fields on organizations / users — Phase 2 after email/password.

15. What “best in class” means for NEXUS
Single contract layer — Zod shared across web, API, and OpenAPI.
Strict modular monolith — your controller/routes/validators/services/repository pattern; extract microservices only if needed later.
Tenant-safe by default — RLS + middleware double enforcement.
AI as audited workflows — not a chat black box; every action traceable in audit_logs and incident_timeline.
Progressive frontend migration — TanStack Query loaders per route; mocks remain until each module ships.
Schema-first — fix SCHEMA.sql duplication once, then migrations are the only DDL path.
